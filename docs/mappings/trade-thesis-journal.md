# Trade Thesis Journal layer mapping

Status vocabulary: mature, partial, missing, not applicable.

| Layer | Status | Evidence and gap |
| --- | --- | --- |
| Identity | partial | The project should serve investment reasoning discipline, but the exact audience, risk boundaries, and decision cadence need to be written first. |
| Context | partial | Trade ideas, market notes, and thesis snapshots are likely inputs; a canonical context packet format is missing. |
| Skills | partial | Candidate skills include thesis capture, counterargument generation, post-trade review, and position sizing notes. They should be added one at a time. |
| Memory | missing | Durable lessons from past trades need an explicit schema and retrieval path before the journal becomes useful. |
| Connections | partial | Market-data and broker connections are tempting but should start as manual/local imports to avoid premature automation. |
| Verification | missing | Verification should include pre-trade checklist tests, hindsight-bias checks, and retrospective completeness checks before live automation. |
| Automations | missing | Scheduled reminders or data pulls should wait until Identity and Verification mature. |

## Sequencing guidance

Start with Identity and Verification issues: define what decisions the journal may influence, then require every thesis to include assumptions, disconfirming evidence, and review dates.
